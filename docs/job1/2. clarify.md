# 목표

## 요약

- 현실에서 진행할 수 있는 미션 기반 게임 MVP를 만든다.
- 참가자 행동, 선택, 감정, 물건 등을 복선으로 수집하고 마지막 엔딩에서 회수하는 경험을 제공한다.
- 안전하고 민폐가 없는 미션을 우선하며, 단순 관광 추천이 아닌 이야기형 플레이 경험을 목표로 한다.
- 이번 작업 범위에는 웹서비스를 실제로 서비스할 수 있도록 실행, 빌드, 배포 준비 설정을 포함한다.
- 환경 태그는 사용자에게 한국어로 제공하고, 문자열 직접 입력이 아니라 선택/삭제 가능한 태그 박스 UI로 제공한다.
- 게임 시작 후에는 설정/목록 중심 화면을 줄이고 현재 플레이에 필요한 화면만 노출해 몰입감을 높인다.
- 현재 UI의 시각적 완성도를 높여 게임 서비스로 보이는 화면으로 개선한다.

## 상세

| 구분 | 목표 |
|---|---|
| 제품 목표 | “Reality Mission Engine”의 MVP를 구현 가능한 수준으로 구체화하고 제작한다. |
| 경험 목표 | 참가자들이 약 6시간 30분 동안 미션을 수행하며 함께 웃고, 마지막에 수집된 복선이 하나의 이야기로 연결되는 경험을 만든다. |
| 운영 목표 | 준비된 미션 데이터와 환경 태그를 기반으로 진행하며, AI 호출은 필요한 시점에 제한적으로 사용한다. |
| 안전 목표 | 불법, 위험, 민폐, 창피함 강요, 의미 없는 이동을 배제한다. |
| 서비스 목표 | 로컬 개발 실행뿐 아니라 프로덕션 빌드와 배포 환경 설정이 가능한 웹서비스 형태로 만든다. |
| 입력 경험 목표 | 환경 태그는 한국어 선택지에서 고르고, 선택된 태그는 박스로 표시하며 클릭으로 제거할 수 있게 한다. |
| 몰입 목표 | 게임 시작 후에는 시작 설정, 미션 목록 같은 준비성 정보를 숨기고 현재 미션과 진행 행동에 집중하게 한다. |
| UI 목표 | 기존 기본 패널형 화면을 게임 콘솔에 맞는 시각 언어로 개선한다. |

## 성공 기준

| 기준 | 성공 상태 |
|---|---|
| 플레이 흐름 | 설정, 진행, 체크포인트, 미션 완료, 엔딩, 종료 흐름이 끊기지 않는다. |
| 미션 선택 | 환경 태그와 상태를 기준으로 적절한 미션을 선택할 수 있다. |
| 복선 수집 | 미션 수행 중 사진, 영수증, 색, 대화, 물건, 선택, 감정 같은 복선 정보를 기록할 수 있다. |
| 엔딩 생성 | 수집된 복선을 바탕으로 마지막 이야기형 엔딩을 생성할 수 있다. |
| 안전성 | 금지된 미션 유형과 위험한 행동을 배제하는 규칙이 명확하다. |
| MVP 단순성 | 초기 버전은 JSON 기반 데이터로 동작하며, 복잡한 데이터베이스나 추천 시스템은 후속 버전으로 둔다. |
| 서비스 준비 | 개발 서버 실행, 프로덕션 빌드, 환경 변수 예시, 배포 플랫폼 설정 파일 또는 안내가 준비된다. |
| 한국어 태그 | 화면에는 `walkable`, `indoor` 같은 내부 키가 아니라 한국어 라벨이 표시된다. |
| 태그 선택 UI | 환경 태그를 셀렉트박스에서 선택해 추가하고, 선택된 태그 박스를 클릭해 제거할 수 있다. |
| 플레이 몰입 | 게임 시작 후에는 현재 미션, 복선 입력, 진행 버튼, 진행 상태, 엔딩처럼 플레이에 필요한 영역만 우선 노출된다. |
| UI 개선 | 모바일과 데스크톱에서 텍스트 겹침 없이 게임 분위기에 맞는 화면 구성이 제공된다. |

# 범위

## 요약

- MVP 범위는 JSON 기반 콘텐츠, 환경 태그, 미션 데이터, 게임 상태, 복선 기록, 엔딩 생성이다.
- MVP 범위에는 실제 웹서비스 운영을 위한 프로젝트 설정, 빌드 설정, 배포 준비 설정이 포함된다.
- 후속 범위는 관리자 미션 편집기, 데이터베이스 확장, 로그 분석, 추천 시스템, 사용자 제작 미션이다.
- 단순 여행 추천, 맛집 추천, 관광 코스 추천은 범위에서 제외한다.

## 상세

| 포함 범위 | 설명 |
|---|---|
| 게임 상태 흐름 | SETUP, PLAYING, CHECKPOINT, MISSION_COMPLETE, ENDING, FINISHED, EMERGENCY_MISSION 흐름을 다룬다. |
| 미션 데이터 | 미션 제목, 설명, 태그, 예상 시간, 비용, 단계, 카테고리, 복선 태그를 관리한다. |
| 환경 태그 | 현재 장소와 상황에 맞는 미션 선택을 위해 환경 태그를 사용한다. |
| 복선 기록 | 플레이 중 남긴 선택과 관찰값을 엔딩 생성에 활용할 수 있게 기록한다. |
| AI 활용 | 게임 시작, 챕터 전환, 엔딩 생성, 준비된 분기가 없을 때 AI를 호출한다. |
| 기본 화면 | 플레이어가 현재 상태, 미션, 체크포인트, 복선, 엔딩을 확인하고 진행할 수 있는 화면을 포함한다. |
| 서비스 설정 | 웹 앱 실행, 빌드, 환경 변수 관리, 배포 준비에 필요한 설정과 문서를 포함한다. |
| 환경 태그 UI | 한국어 태그 선택지, 선택된 태그 박스, 태그 제거 인터랙션을 포함한다. |
| 플레이 화면 개선 | 게임 시작 전과 시작 후 화면 상태를 분리하고, 시작 후에는 플레이 관련 화면만 제공한다. |
| 시각 디자인 개선 | 현재 컴포넌트와 CSS 범위 안에서 전체 UI를 개선한다. |

| 제외 범위 | 설명 |
|---|---|
| 고도화된 추천 시스템 | MVP 이후 v1 범위로 둔다. |
| 관리자 미션 편집기 | MVP 이후 v0.5 범위로 둔다. |
| SQLite 또는 PostgreSQL 기반 저장소 | MVP에서는 JSON 기반으로 시작한다. |
| 사용자 제작 미션 | MVP 이후 v1 범위로 둔다. |
| 단순 관광/맛집 추천 | 제품 정체성과 맞지 않아 제외한다. |

# 미결 사항

## 요약

- 현재 미결 사항은 MVP 구현을 막는 차단 요소는 아니다.
- 실제 AI API 연동 여부와 초기 미션 데이터 규모는 설계와 명세 단계에서 구체화가 필요하다.
