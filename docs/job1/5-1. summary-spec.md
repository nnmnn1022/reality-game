# 전체 요약

## 요약

- 현실에서 플레이하는 미션 게임 웹서비스 MVP를 만든다.
- 참가자는 미션을 수행하며 복선을 남기고, 마지막에는 그 복선이 이야기형 엔딩으로 회수된다.
- 이번 작업은 화면 구현뿐 아니라 실제 서비스 실행과 배포 준비 설정까지 포함한다.

## 상세

| 구분 | 내용 |
|---|---|
| 제품 성격 | 여행 추천이 아니라 현실에서 플레이 가능한 이야기형 미션 게임이다. |
| 핵심 경험 | 처음엔 의미 없어 보이는 행동이 마지막에 하나의 이야기로 연결된다. |
| 서비스 형태 | 브라우저에서 실행되는 웹서비스로 제공한다. |
| 운영 방식 | 초기 버전은 준비된 미션 데이터와 브라우저 저장 상태를 사용한다. |

# 문제

## 요약

- 미션이 재미있어도 위험하거나 민폐가 되면 안 된다.
- 단순 장소 추천이 아니라 참가자가 직접 행동하고 함께 웃는 구조가 필요하다.
- 마지막 엔딩이 플레이 중 남긴 흔적과 연결되어야 한다.

## 상세

| 문제 | 설명 |
|---|---|
| 안전성 | 위험, 불법, 민폐, 창피함 강요를 배제해야 한다. |
| 참여성 | 모든 참가자가 함께 행동하고 선택할 수 있어야 한다. |
| 이야기성 | 복선이 누적되고 엔딩에서 회수되어야 한다. |
| 서비스성 | 개발자 PC에서만 동작하는 수준이 아니라 배포 가능한 웹서비스여야 한다. |

# 원인

## 요약

- 신규 제품 제작이므로 장애 원인은 없다.
- 현재는 기획 문서만 있고 실제 실행 가능한 서비스가 없다.

## 상세

| 항목 | 내용 |
|---|---|
| 작업 유형 | 신규 MVP 제작 |
| 현재 한계 | 화면, 상태 관리, 데이터, 서비스 설정이 아직 없다. |
| 해결 방향 | 기획 내용을 실행 가능한 웹서비스 구조로 옮긴다. |

# 현재(AS-IS)

## 요약

- 작업 루트에는 기획 문서와 워크플로 문서가 있다.
- 아직 사용자가 접속해 플레이할 수 있는 앱은 없다.
- 배포나 운영을 위한 설정도 아직 없다.

## 상세

| 영역 | 현재 상태 |
|---|---|
| 기획 | 현실 미션 게임의 방향이 문서로 정리되어 있다. |
| 실행 | 브라우저에서 실행되는 서비스가 없다. |
| 저장 | 플레이 상태와 복선 기록 저장 방식이 구현되어 있지 않다. |
| 배포 | 서비스 배포를 위한 설정과 안내가 없다. |

# 목표(TO-BE)

## 요약

- 사용자가 웹에서 게임을 시작하고 미션을 진행할 수 있게 한다.
- 환경에 맞는 미션을 선택하고, 완료한 미션과 복선을 누적한다.
- 누적된 복선으로 엔딩을 생성한다.
- 개발 실행, 빌드, 배포 준비가 가능한 서비스 형태로 만든다.

## 상세

| 목표 | 성공 상태 |
|---|---|
| 게임 시작 | 참가자와 환경 태그를 입력하고 첫 미션을 받을 수 있다. |
| 미션 진행 | 현재 미션을 확인하고 완료 처리할 수 있다. |
| 복선 기록 | 선택, 감정, 관찰, 물건 같은 기록을 남길 수 있다. |
| 엔딩 | 누적 기록이 포함된 이야기형 엔딩을 볼 수 있다. |
| 서비스 준비 | 실행 방법, 빌드 방법, 환경 설정 예시, 배포 설정이 준비된다. |

# 그 외 참고

## 요약

- 실제 AI API가 없어도 MVP는 동작해야 한다.
- 데이터베이스, 관리자 편집기, 고도화 추천은 후속 범위다.

## 상세

| 항목 | 방향 |
|---|---|
| AI | 기본은 로컬 생성이며, 실제 AI 연결은 환경 설정으로 확장 가능하게 둔다. |
| 저장 | 초기 버전은 브라우저 저장을 사용한다. |
| 후속 확장 | 관리자 편집기, 데이터베이스, 분석, 추천 시스템은 이후 단계로 둔다. |
