# 문제

## 요약

- 현실에서 플레이 가능한 AI 기반 미션 게임을 만들고자 한다.
- 단순 여행 추천이나 관광 안내가 아니라, 참가자의 행동과 선택이 마지막에 하나의 이야기로 연결되는 경험이 필요하다.
- 안전, 재미, 복선, 참여를 해치지 않으면서 현실 장소와 상황에 맞는 미션 흐름을 구성해야 한다.

## 상세

| 항목 | 내용 |
|---|---|
| 사용자 요청 | 첨부 기획서를 참고해 현실 게임 제작을 차근차근 진행한다. |
| 작업 루트 | 사용자 제공 경로인 `/home/umoo/reality-game`을 사용한다. |
| 핵심 문제 | 현실에서 수행 가능한 미션을 통해 참가자들이 함께 웃고, 마지막에는 수집된 복선이 하나의 이야기로 회수되는 게임 경험을 만들어야 한다. |
| 피해야 할 방향 | 민폐, 불법, 위험, 단순 관광 추천, 의미 없는 이동, 창피함 유도, 검색이 필요한 미션을 피해야 한다. |
| 산출 방향 | 기획서 기반으로 개발 워크플로우를 순서대로 진행하고, 각 단계의 산출물을 작업 루트 하위 문서 디렉터리에 남긴다. |

# 맥락

## 요약

- 첨부 기획서는 “Reality Mission Engine”을 AI 여행 추천 서비스가 아닌 현실 플레이형 이야기 연출 엔진으로 정의한다.
- 플레이는 약 6시간 30분 동안 진행되며, 약 20분마다 체크포인트를 둔다.
- 미션은 예능성, 감성/복선, 휴식을 섞고, 마지막 엔딩은 수집된 복선을 AI가 회수하는 방식으로 구상되어 있다.

## 상세

| 구분 | 기획서 기준 맥락 |
|---|---|
| 핵심 경험 | 처음에는 의미 없어 보이는 행동이 마지막에 하나의 이야기로 연결된다. |
| 핵심 가치 | 안전, 재미, 복선, 참여, 효율 순으로 중요하게 본다. |
| 플레이 흐름 | 큰 미션과 작은 미션을 섞고, 약 20분마다 체크포인트를 둔다. |
| 스토리 접근 | 결말을 미리 작성하지 않고 복선을 수집한 뒤 마지막에 회수한다. |
| 미션 성격 | 예능성 미션 65%, 감성/복선 미션 25%, 휴식 10%를 목표로 한다. |
| 좋은 미션 조건 | 어렵지 않고, 함께 웃을 수 있으며, 환경에 맞고, 5~15분 안에 수행되며, 복선을 남긴다. |
| 복선 수집 대상 | 사진, 영수증, 색, 대화, 물건, 선택, 감정 등을 활용한다. |
| MVP 방향 | JSON 기반, 환경 태그, 미션 데이터, 복선, 엔딩 생성을 우선한다. |

## 범위 경계

| 포함 | 제외 |
|---|---|
| 첨부 기획서 기반의 현실 미션 게임 MVP 구체화 | 단순 여행 추천 서비스 |
| 안전하고 참여 가능한 미션 흐름 정의 | 위험하거나 불법적인 행동 유도 |
| 복선 수집과 엔딩 회수 경험 설계 | 참가자에게 민폐나 창피함을 강요하는 미션 |
| 작업 루트 내 문서화와 단계별 개발 진행 | 단계 생략 또는 명세 없이 바로 구현 |
