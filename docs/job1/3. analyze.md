# 분석

## 요약

- 현재 MVP는 Next.js 앱으로 구현되어 있고 `https://reality-game.umoo.pe.kr/`에서 Caddy 리버스 프록시로 서비스된다.
- 화면은 `GameConsole` 단일 컴포넌트 중심이며, 설정/현재 미션/진행/엔딩/상태/미션 목록/복선이 한 화면에 함께 노출된다.
- 환경 태그는 콤마로 구분된 문자열 입력이며, 기본값과 내부 데이터는 영문 태그 키를 사용한다.
- 이번 변경의 영향 범위는 태그 표시/입력 UI, 게임 시작 전후 화면 분기, CSS 시각 개선, 관련 테스트와 빌드 검증이다.

## 영향 범위

| 범위 | 영향 |
|---|---|
| 데이터 | 미션 목록, 현재 게임 상태, 복선 기록을 JSON으로 다룰 필요가 있다. |
| 상태 흐름 | 설정, 진행, 체크포인트, 미션 완료, 예외 미션, 엔딩, 종료 상태가 사용자 조작과 연결되어야 한다. |
| 콘텐츠 규칙 | 안전, 민폐 방지, 환경 태그 적합성, 복선 다양성 규칙이 미션 선택에 반영되어야 한다. |
| AI 경계 | AI는 무조건 새 미션을 만드는 역할이 아니라 기존 미션 선택, 연출, 엔딩 생성 중심으로 제한된다. |
| 사용자 경험 | 플레이어가 현재 미션, 진행 상태, 복선, 엔딩을 이해하고 조작할 수 있어야 한다. |
| 서비스 준비 | 웹서비스로 실행 가능한 프로젝트 설정, 프로덕션 빌드, 환경 변수 예시, 배포 준비 설정이 필요하다. |
| 검증 | 상태 전이, 미션 필터링, 복선 기록, 엔딩 생성 경로를 확인할 수 있어야 한다. |
| 태그 표시 | 내부 영문 태그 키는 유지하되 사용자에게는 한국어 라벨을 표시해야 한다. |
| 입력 UI | 콤마 문자열 입력을 셀렉트박스와 제거 가능한 태그 박스로 대체해야 한다. |
| 화면 전환 | SETUP 상태와 플레이 진행 상태의 화면 구성을 분리해야 한다. |
| 스타일 | 현재 CSS 전반을 수정해 게임 화면에 맞는 밀도, 색상, 레이아웃을 제공해야 한다. |
