# 개발

## 요약

- 현실 미션 게임 웹서비스 MVP의 실행 뼈대를 구현했다.
- 샘플 미션 데이터는 최소 수준으로만 두고, 이후 사용자가 직접 추가하기 쉽게 JSON 구조로 정리했다.
- 개발 실행, 타입 검사, 린트, 테스트, 프로덕션 빌드, 배포 설정까지 서비스 준비에 필요한 파일을 추가했다.

## 상세

| 구분 | 내용 |
|---|---|
| 앱 구조 | Next.js App Router 기반 단일 페이지 웹서비스로 구성했다. |
| 상태 관리 | 설정, 진행, 체크포인트, 완료, 범용 미션, 엔딩, 종료 상태를 순수 함수로 관리했다. |
| 데이터 | `data/missions.json`에 최소 샘플 미션을 넣고, 구조는 이후 직접 확장 가능하게 만들었다. |
| 저장 | 브라우저 `localStorage`로 게임 상태를 저장하고 복원한다. |
| 초기 저장 안전성 | 첫 하이드레이션 이전에는 저장하지 않도록 막아 기존 상태를 덮어쓰지 않게 했다. |
| 서비스 준비 | `.env.example`, `vercel.json`, `README.md`를 추가해 실제 서비스 준비 경로를 열었다. |
| 테스트 | 상태 전이와 엔딩 생성 핵심 동작을 테스트로 묶었다. |
