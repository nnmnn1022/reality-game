# FLOW_SYSTEM

## Definition

Flow는 Experience의 리듬과 균형을 정의한다.  
Flow는 Story를 쓰지 않는다.

## Flow Components

### Target
Flow가 만들고자 하는 Experience의 방향.

예:
- Chaos Trip: 예측 불가능, 웃음, 당황, Callback
- Mystery: 질문, 발견, 혼란, Reveal, Resolution
- Adventure: 탐험, 발견, 도전, Peak, 회상
- Bond: 대화, 협력, 이해, 기억

### Coverage Definition
해당 Flow에서 충족되어야 하는 Experience 요소.

예: Adventure Flow
- Exploration
- Discovery
- Challenge
- Reflection

예: Comedy Flow
- Awkward
- Chaos
- Callback
- Group Laughter

### Stage Graph
Stage들의 권장 전환 구조. 선형일 수도 있고 그래프일 수도 있다.

### Balancer
현재 Coverage와 Event를 보고 다음 Stage를 선택하는 원칙.

## Important Principle

Graph는 깨질 수 있다.  
Target은 유지되어야 한다.

플레이어가 예상 밖의 행동을 하면 Engine은 Flow를 강제하지 않고 Coverage의 균형을 다시 맞춘다.
