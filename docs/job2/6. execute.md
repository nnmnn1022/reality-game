# 개발

## 요약

- 웹 콘솔 중심 구현을 제거하고, 디스코드 Interaction Endpoint 기반 게임 백엔드로 바꿨다.
- 세션 상태는 디스크 저장소에 저장하고, 미션 선택/완료/체크포인트/엔딩/종료를 interaction 핸들러에서 처리한다.
- 루트 페이지는 상태 확인용으로만 남기고, 실제 플레이는 디스코드에서 진행되도록 바꿨다.

## 상세

| 변경 파일 | 핵심 내용 |
|---|---|
| `src/lib/game.ts` | 게임 상태와 미션 선택, 완료, 체크포인트, 엔딩, 종료, 직렬화/복원 로직을 디스코드 세션용으로 재구성했다. |
| `src/lib/discord-interactions.ts` | Discord command, button, modal submit을 처리하고 세션 상태를 갱신하는 핸들러를 추가했다. |
| `src/lib/session-store.ts` | 세션을 JSON 파일로 저장/복원하는 저장소를 추가했다. |
| `src/app/api/discord/interactions/route.ts` | Discord Interaction Endpoint를 구현하고 서명 검증과 JSON 파싱 실패를 처리한다. |
| `src/types/game.ts` | 디스코드 세션과 interaction 처리에 필요한 상태 타입을 확장했다. |
| `src/app/page.tsx` | 웹 콘솔을 제거하고 상태 확인용 페이지로 교체했다. |
| `src/app/layout.tsx` | 메타데이터를 디스코드 백엔드 기준으로 수정했다. |
| `src/app/globals.css` | 기존 패널형 웹 UI 스타일을 제거하고 상태 페이지 스타일로 교체했다. |
| `src/test/game.test.ts` | 세션 시작, 복선 기록, 디스코드 interaction 시작/모달 완료를 검증하도록 테스트를 교체했다. |
| `README.md` | 웹앱 안내를 디스코드 백엔드 안내로 수정하고 명령 목록과 환경 변수를 문서화했다. |
| `.env.example` | Discord 공개키, 서명 검증 스킵, 저장 경로 예시를 추가했다. |
| `vercel.json` | 웹 배포 전용 설정을 제거했다. |
| `src/components/GameConsole.tsx` | 웹 콘솔 컴포넌트를 제거했다. |
