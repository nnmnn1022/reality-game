# 목표

## 요약

- `reality-game`의 기존 웹앱 구현을 폐기하고 디스코드 기반 게임을 새로 만든다.
- 제품의 핵심은 플레이어 입력과 선택이 누적되어 하나의 이야기로 회수되는 경험이다.
- 이번 단계에서는 디스코드 UX 세부가 아직 없어도, 제품 목표와 범위를 먼저 고정한다.
- 디스코드 상호작용 방식은 추후 문서가 추가되면 세부화한다.

## 상세

| 항목 | 내용 |
|---|---|
| 전환 목표 | 기존 웹앱 구현을 재사용하지 않고, 디스코드에서 플레이하는 게임을 새로 만든다. |
| 경험 목표 | 단발성 정보 제공이 아니라, 참여와 선택이 쌓여 서사로 회수되는 경험을 만든다. |
| 진행 방식 | 현재 제공된 기획 문서를 기준으로 제품 목표와 범위를 정리하고, 기존 웹앱은 새 구현의 기준이 아니라 폐기 대상으로 본다. |
| 임시 가정 | 디스코드 UX 세부는 아직 미정이므로, 이번 워크플로우는 고수준 제품 목표와 범위만 먼저 확정하는 것으로 진행한다. |

# 범위

## 요약

- 포함: 디스코드 게임으로의 제품 전환, 핵심 경험 정의, 현재 작업의 경계 설정
- 포함: 추후 설계와 구현을 위한 기준 문서 정리
- 제외: 디스코드 메시지 흐름, 버튼 구조, 알림 방식 같은 세부 UX 확정
- 제외: 구현 코드, 상세 설계, 상태 전이 규칙, API/봇 구조

## 상세

| 구분 | 포함 | 제외 |
|---|---|---|
| 제품 방향 | 디스코드 게임으로 새로 구축 | 웹앱 전제 유지 |
| 경험 정의 | 입력과 선택이 서사로 연결되는 게임 경험 | 채널별 UX 세부 |
| 작업 경계 | 다음 단계로 넘길 기준 정리 | 기존 웹앱 코드 보존 여부 판단 |
| 추가 입력 | 후속 UX 문서를 반영할 준비 | 현재 단계에서 미확정 내용을 결정 |

# 미결 사항

## 요약

- 디스코드 UX 세부는 아직 제공되지 않았다.
- 봇 중심 상호작용의 입력 방식, 진행 안내, 알림 정책은 후속 문서가 필요하다.

## 상세

| 항목 | 상태 | 영향 |
|---|---|---|
| 디스코드 UX 세부 | 미제공 | 상세 설계와 구현 방식에 영향 |
| 입력 방식 | 미정 | 플레이 흐름과 조작성을 좌우함 |
| 진행 안내 | 미정 | 사용자 이해도와 이탈률에 영향 |
| 알림 정책 | 미정 | 게임 몰입과 소음 수준에 영향 |
