# 문제

## 요약

- 기존에 웹앱으로 구현되어 있던 `reality-game`을 폐기하고, 디스코드 기반 게임을 새로 만들어야 한다.
- 첨부된 기획 문서 묶음은 게임의 핵심 개념과 엔진 방향을 담고 있지만, 디스코드에서의 사용자 흐름과 상호작용 규칙은 아직 부족하다.
- 이번 작업은 기존 웹앱 구현을 새 출발점으로 삼지 않고, 디스코드 게임 기준으로 다시 설계하고 구현 가능한 형태로 정리하는 출발점이다.

## 상세

| 항목 | 내용 |
|---|---|
| 사용자 요청 | 기존 웹앱을 지우고, 디스코드 게임을 새로 만드는 기준으로 전체 작업을 진행한다. |
| 기준 자료 | 사용자 제공 압축 문서에 게임의 핵심 제품 정의, 상태 개념, 미션 구조, 엔진 역할이 정리되어 있다. |
| 핵심 문제 | 웹서비스 전제의 기존 구현을 버리고, 디스코드 환경에 맞는 게임 경험으로 다시 정의해야 한다. |
| 현재 부족한 점 | 디스코드에서의 UX, 입력 방식, 진행 안내, 상호작용 규칙은 추가 문서가 더 필요하다. |
| 작업 방식 | 개발 워크플로우를 순서대로 진행하고, 기존 웹앱을 재사용 대상으로 보지 않은 상태에서 다음 단계로 넘어간다. |

# 맥락

## 요약

- 제품은 단순 추천이나 정보 제공이 아니라, 플레이어 입력과 선택이 누적되어 하나의 이야기로 회수되는 경험을 목표로 한다.
- 엔진은 이야기 자체를 강제로 만들기보다, 진행 흐름과 경험의 균형을 잡는 역할로 정의되어 있다.
- 스토리는 미리 고정된 결말이 아니라, 플레이어가 남긴 사건과 흔적을 바탕으로 형성된다.
- 미션은 상호작용 방식, 제약, 입력을 묶은 단위로 이해하는 편이 맞다.

## 상세

| 구분 | 맥락 |
|---|---|
| 제품 성격 | 플레이어가 직접 참여하고 선택하는 디스코드 게임형 경험이다. |
| 핵심 역할 | 시스템은 진행 흐름과 경험 균형을 관리하고, 이야기의 최종 형성은 플레이어 입력에 크게 의존한다. |
| 구조적 개념 | 경험, 흐름, 단계, 스토리 비트, 미션, 입력, 결과, 스토리 기억, 서사라는 계층으로 이해할 수 있다. |
| 상태 개념 | 스토리 비트는 준비, 활성, 해결의 흐름을 가진다. |
| 미션 개념 | 미션은 상호작용 방식과 제약, 플레이어 입력이 결합된 단위다. |
| 운영 맥락 | 봇이 입력을 수집하고, 사건을 기억으로 정리하며, 최종적으로 스토리로 회수하는 구성이 핵심이다. |
| 전환 맥락 | 기존 웹앱 구현을 폐기하고, 디스코드 대화형 게임 경험으로 새로 만드는 것이 이번 작업의 중심이다. |

|---|---|

| 다음 단계에서 사용할 수 있는 작업 맥락 정리 | 기존 웹앱 코드의 유지 여부 판단 |
